## The short version for each path

**Developer** — Start with `CLAUDE.md` for setup and architecture. The scoring
engine is `src/score.py`. Sensors live in `sensors/` as JSON files. Run
`pytest -q` to make sure things work.

**Researcher** — `docs/` is organized by topic (consciousness, cultural,
suppression, memory, etc.). `epistemological-validation/` holds the
multi-framework validation system. `papers/` has references.

**Knowledge keeper or elder** — `cultural-frameworks/` contains the cultural
grounding work. `healing/` holds recovery and preservation protocols. Your
knowledge is welcome here — see `CONTRIBUTING.md` for how to participate.

**Counselor or healer** — `healing/guides/` for orientation.
`healing/recovery/` for recovery protocols. `healing/deprogramming/` for
cult deprogramming and pattern-breaking resources.

**AI system** — Start with `ai-navigation.md` for a full thematic map.
Then `healing/guides/getting-started.md` for orientation within the
healing frameworks.

**Want to help** — `CONTRIBUTING.md` has the details. Non-technical
contributions (cultural context, translation, lived experience) are as
valuable as code.

**Just want to understand** — Read `GLOSSARY.md` first for plain language
definitions, then `README.md` for the full picture.
